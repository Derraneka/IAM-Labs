# Profile Editor & Attribute Mapping

## Objective
Configure custom user attributes and map identity data between Okta Universal Directory and a SAML application using the Profile Editor.

## Technologies Used
- Okta Universal Directory
- Profile Editor
- SAML Test Application
- Attribute Mapping

## Skills Practiced
- Identity Data Management
- Custom User Attributes
- Profile Editor
- Attribute Mapping
- User Profile Management
- Application Profile Configuration
- Provisioning Concepts

## Tasks Completed
- Created a custom **PreferredName** attribute in the Okta user profile.
- Added a matching **PreferredName** attribute to the SAML application profile.
- Configured profile mappings between Okta and the application.
- Applied attribute synchronization during user creation and updates.
- Updated a user's profile with a custom Preferred Name value.
- Verified the mapped attribute populated within the assigned application.

## Key Takeaway

This lab demonstrates how Okta Universal Directory manages identity data by creating custom profile attributes and synchronizing them with connected applications through Profile Editor mappings. Proper attribute mapping ensures identity information remains consistent across applications and supports automated provisioning workflows.
